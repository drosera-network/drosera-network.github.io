A world with Drosera.

December 22, 2023 · 2 min read
Kratos
CoFounder & Founding Engineer of Drosera
Bobafetador
CoFounder & CTO of Drosera

In the rapidly evolving world of decentralized finance (DeFi), security is paramount. The DeFi ecosystem has experienced numerous exploits and hacks resulting in millions of dollars in losses for various protocols. To address these issues, we present Drosera, a native Ethereum solution designed to provide robust security primitives and foster a more secure and robust DeFi environment.

Understanding Drosera​

Drosera is a trustless and decentralized infrastructure for detecting exploits and mitigating financial loss. It is an automation protocol that abstracts the complexity of creating monitoring systems for decentralized applications. Drosera offers a framework for creating and executing automated responses to events on the Ethereum network, enabling developers to create more secure and robust applications.

The Need for Drosera​

The DeFi ecosystem has witnessed an alarming number of exploits and hacks in recent years. These incidents have not only resulted in significant financial losses but have also undermined the trust and confidence of users in the DeFi space. Drosera was born out of the need for a more robust and decentralized security system for DeFi protocols, one that is more native to the Ethereum ecosystem with a strong foundation that can be built upon.

Simplicity and Developer Friendliness​

Drosera's design prioritizes developer operations and experience. It seamlessly integrates with existing Solidity tooling and continuous integration/continuous deployment (CI/CD) pipelines. This allows developers to create and deploy secure DeFi applications with ease, fostering a more secure and efficient DeFi ecosystem.

Drosera's Unique Capabilities​

Drosera gives smart contracts the ability to analyze a well-formatted history of on-chain data. This allows the detection of invariants that change over time instead of just looking at the current state of the blockchain. This powerful capability enables the creation of more sophisticated analsysis that can detect a wide range of invariant condidtions.

These smart cotracts are called Traps, and they implement two functions, collect and isValid. The collect function is responsible for gathering data from the blockchain and returning a struct with data points to monitor. The isValid function accepts an array of the returned struct and returns a bool. This function is called by the Operator on every new block with the collected data and is used to determine whether or not the Trap response should be executed.
